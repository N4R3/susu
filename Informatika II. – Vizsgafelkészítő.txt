### 2. Mock Log Generator
**Rövid magyarázat:**  
Teszteléshez generál fake Windows-szerű logokat (sikeres/sikertelen belépés, IP, timestamp). Lehetővé teszi a SIEM működésének tesztelését valós logok nélkül is.

**Példa:**
```python
import random
from datetime import datetime, timedelta
def generate_mock_logs(n=200, brute_force=False):
    logs = []
    for i in range(n):
        ip = f"192.168.{random.randint(1,10)}.{random.randint(1,255)}"
        if brute_force and i < 15:
            ip = "185.22.33.44"  # támadó IP
        status = "failed" if random.random() < 0.3 else "success"
        ts = datetime.now() - timedelta(minutes=random.randint(0,60))
        logs.append(f"[{ts}] LOGIN {status} FROM {ip} EVENTID 4625")
    return logs
```

**Mit kell tudnom elmagyarázni?**  
- Miért kell mock data a fejlesztéshez és unit tesztekhez.  
- Hogyan szimulálunk támadást (brute force cluster).  
- Paraméterezhetőség és reprodukálhatóság (random.seed).

**Mit kérdezhetnek / mit kellhet módosítani?**  
- Generálj csak brute force logokat 5 percen belül.  
- Adj hozzá fájl-módosítási eventet.  
- Tedd determinisztikussá seed-del.

### 3. Windows Log Parser
**Rövid magyarázat:**  
Beolvassa és strukturálja a log sorokat (timestamp, IP, event type, user). Regex vagy csv.DictReader-rel dolgozik. Kinyeri a SOC szempontból fontos mezőket.

**Példa:**
```python
import re
def parse_log_line(line):
    pattern = r'\[(.*?)\] LOGIN (\w+) FROM (\S+) EVENTID (\d+)'
    match = re.match(pattern, line)
    if match:
        return {
            'timestamp': match.group(1),
            'status': match.group(2),
            'ip': match.group(3),
            'event_id': int(match.group(4))
        }
    return None
```

**Mit kell tudnom elmagyarázni?**  
- Log formátum feldolgozása (regex vs. struktúrált).  
- Milyen mezőket kell kinyerni SOC-hoz (IP, status, event_id).  
- Hibakezelés rossz soroknál.

**Mit kérdezhetnek / mit kellhet módosítani?**  
- Írd át a parsert, hogy Event ID 4625-re fókuszáljon.  
- Adj hozzá user mezőt.  
- Kezeld a hiányzó mezőket gracefully.

### 4. Anomaly Detector
**Rövid magyarázat:**  
Egyszerű szabályalapú detektor: számolja a sikertelen belépéseket IP-nként időablakban és jelzi a brute force-t.

**Példa:**
```python
from collections import defaultdict
def detect_anomalies(logs, threshold=5, window_minutes=5):
    failed = defaultdict(list)
    for log in logs:
        if log['status'] == 'failed':
            failed[log['ip']].append(log['timestamp'])
    suspicious = [ip for ip, times in failed.items() if len(times) >= threshold]
    return suspicious
```

**Mit kell tudnom elmagyarázni?**  
- Threshold + időablak logika (SIEM szabályok).  
- Brute force felismerés.  
- Miért egyszerű szabályalapú (nem ML) egy BSc projektben.

**Mit kérdezhetnek / mit kellhet módosítani?**  
- Emeld a thresholdöt 10-re és adj hozzá időablak szűrést.  
- Integrálj egyszerű ML-t (pl. Isolation Forest) – de csak ha kérdezik.

### 5. Integrity Checker (HIDS)
**Rövid magyarázat:**  
Fájlok SHA256 hash-ét számítja, baseline JSON-nal összehasonlítja → változás = riasztás (klasszikus HIDS).

**Példa:**
```python
import hashlib, json, os
def calculate_hash(file_path):
    with open(file_path, 'rb') as f:
        return hashlib.sha256(f.read()).hexdigest()

def check_integrity(baseline_path, files_to_check):
    with open(baseline_path) as f:
        baseline = json.load(f)
    changes = []
    for file in files_to_check:
        current = calculate_hash(file)
        if current != baseline.get(file):
            changes.append(file)
    return changes
```

**Mit kell tudnom elmagyarázni?**  
- Mi a HIDS és miért kell fájlintegritás-ellenőrzés.  
- Hash funkció (sha256 miért jó).  
- Baseline vs. current összehasonlítás.

**Mit kérdezhetnek / mit kellhet módosítani?**  
- Adj hozzá új fájlt a baseline-hez.  
- Változtasd MD5-re (és magyarázd miért rosszabb).  
- Implementálj rekurzív mappakezelést.

### 2. SIEM logelemzés

**Rövid magyarázat**  
SIEM (Security Information and Event Management) gyűjti, tárolja és elemzi a különböző rendszerek logjait, hogy korrelált, strukturált eseményekként mutassa a biztonsági eseményeket. A logelemzés részében keresünk ismétlődő, gyanús mintákat. [negg](https://negg.blog/en/siem-and-cyber-security-advanced-protection-and-monitoring/)

**Kulcsfogalmak**

- Normalizálás (log struktúra),  
- Korrelációs szabály,  
- Esemény súlyozás (severity),  
- Dashboards / alerting. [freecodecamp](https://www.freecodecamp.org/news/how-to-create-a-python-siem-system-using-ai-and-llms/)

**Vizsgán hogyan kérdezhetik**  
- „Miért fontos a log normalizálás?”  
- „Írj egy példa szabályt, amely több hibás bejelentkezést számol meg.”  

**Python‑példa (nagyon egyszerű SIEM‑szerű loggyűjtés)**  

```python
import datetime
import json

LOG_FILE = "siem_logs.json"

def log_event(source, event_type, ip, message):
    entry = {
        "timestamp": datetime.datetime.now().isoformat(),
        "source": source,
        "event_type": event_type,
        "ip": ip,
        "message": message
    }
    with open(LOG_FILE, "a", encoding="utf-8") as f:
        f.write(json.dumps(entry) + "\n")

# Használat:
log_event("Windows Firewall", "BLOCK", "192.168.1.100", "Rendszerfelület elutasítva")
```

**Gyakori hibák**  
- Nem timestampelt, „nyers” szövegként ír logot, nem JSON/strukturáltan.  
- Nem figyel a log típusra (pl. minden esemény „INFO” marad).

### 3. HIDS / fájlintegritás‑ellenőrzés

**Rövid magyarázat**  
HIDS (Host‑based Intrusion Detection System) a gépen belül figyeli a fájlok, regiszterek, folyamatok változásait. Fájlintegritás‑ellenőrzés: egy „baseline” hash alapján vizsgáljuk, módosult-e a fájl. [github](https://github.com/cedricbonhomme/pyHIDS)

**Kulcsfogalmak**  
- Hash (pl. SHA‑256),  
- Baseline / referencia adatbázis,  
- Fájlváltozás (integritási sértés),  
- Signature detection (pl. „HASH változott”),  
- Wazuh / Open‑source HIDS (pl. pyHIDS). [runebook](https://runebook.dev/en/docs/python/library/hashlib/file-hashing)

**Vizsgán hogyan kérdezhetik**  
- „Miért nem elég a dátumváltozás alapján figyelni egy fájlt?”  
- „Mi az alapelve a hash‑alapú integritásellenőrzésnek?”  

**Python‑példa (fájlhash tárolása és ellenőrzése)**  

```python
import hashlib
import json
import os

DB_FILE = "hash_db.json"

def compute_hash(filepath):
    h = hashlib.sha256()
    with open(filepath, "rb") as f:
        while chunk := f.read(4096):
            h.update(chunk)
    return h.hexdigest()

def save_baseline(filelist):
    db = {}
    for fp in filelist:
        if os.path.exists(fp):
            db[fp] = compute_hash(fp)
    with open(DB_FILE, "w") as f:
        json.dump(db, f, indent=2)

def check_integrity():
    if not os.path.exists(DB_FILE):
        print("Nincs baseline adatbázis.")
        return
    with open(DB_FILE) as f:
        db = json.load(f)
    for fp, old_hash in db.items():
        if not os.path.exists(fp):
            print(f"!! FÁJL ELVESZETT: {fp}")
            continue
        new_hash = compute_hash(fp)
        if new_hash != old_hash:
            print(f"!! INTEGRITÁS VISSZAESÉS: {fp}")
```

**Gyakori hibák**  
- Óriási fájlokat egyben beolvasni, emlékméret‑problémák.  
- Nem kezelni a fájl hiányát vagy jogosultságokat a `FileNotFoundError`/`PermissionError` kezelése nélkül.

    match = re.match(LOG_PATTERN, line.strip())
    if not match:
        return None
    fields = match.groups()
    return {
        "ip": fields[0],
        "timestamp": fields [microsoft](https://www.microsoft.com/hu-hu/security/business/security-101/what-is-a-security-operations-center-soc),
        "method": fields [beos](https://beos.hu/biztonsagi-muveleti-kozpont-soc-definicioja-es-szerepe-az-it-biztonsagban/),

### 5. Python regex alapok loghoz

**Rövid magyarázat**  
A Python `re` modul segítségével regex (szabályos kifejezés) segítségével keresünk, illetve kinyerünk részstringeket logokból. [stackoverflow](https://stackoverflow.com/questions/30956820/log-parsing-with-regex)

**Kulcsfogalmak**  
- `re.match()` / `re.search()`,  
- Capturing group `(...)`,  
- `\S+`, `\d+`, `.*`,  
- Named group `(?P<nev>...)`,  
- Greedy / non‑greedy (`.*?`). [stackoverflow](https://stackoverflow.com/questions/30956820/log-parsing-with-regex)

**Vizsgán hogyan kérdezhetik**  
- „Miben különbözik a `match` és a `search`?”  
- „Mondj egy regex példát, amely IP‑t keres.”  

**Python‑példa (IP‑kinyerés)**  

```python
import re

LOG_LINE = "ERROR 07/May/2026:10:01:00 192.168.1.100: Connection refused"

IP_RE = r"((?:\d{1,3}\.){3}\d{1,3})"

match = re.search(IP_RE, LOG_LINE)
if match:
    print("IP:", match.group(1))
```

**Gyakori hibák**  
- Nem `re.search`‑et használni, ha a minta nem a sor elején van.  
- Túl szigorú regex (pl. rögzített szóközök, amelyek nem stimmelnek).

### 6. Brute force detection egyszerű szabályokkal

**Rövid magyarázat**  
Brute force (pl. SSH/Windows‑bejelentkezés) számlálásához adott IP címről számoljuk a sikertelen kísérleteket egy időablakon belül. [negg](https://negg.blog/en/siem-and-cyber-security-advanced-protection-and-monitoring/)

**Kulcsfogalmak**  
- Sikertelen bejelentkezés (pl. `Login failed`, `AUTHFAIL`),  
- IP‑alapú számlálás,  
- Időablak (pl. 5 perc),  
- Threshold (pl. 5 sikertelen / 1 perc),  
- Alert / logszint emelés. [freecodecamp](https://www.freecodecamp.org/news/how-to-create-a-python-siem-system-using-ai-and-llms/)

**Vizsgán hogyan kérdezhetik**  
- „Milyen logjelölőket keress egy brute‑force szabályban?”  
- „Milyen logikája lehet egy egyszerű brute‑force szabálynak?”  

**Python‑példa (szimpla brute‑force számláló)**  

```python
from datetime import datetime, timedelta
import re

LOG_LINE_RE = r"(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}) .* ip=(\S+) .* Login failed"

alerts = []

def check_bruteforce(log_lines, threshold=3, window=120):
    """
    log_lines: [{ 'timestamp': dt, 'ip': str }]
    """
    now = datetime.now()
    for line in log_lines:
        line_time = datetime.fromisoformat(line["timestamp"])
        if now - line_time > timedelta(seconds=window):
            continue
        for alert in alerts:
            if alert["ip"] == line["ip"]:
                alert["count"] += 1
                if alert["count"] >= threshold:
                    alert["alerted"] = True
                break
        else:
            alerts.append({
                "ip": line["ip"],
                "count": 1,
                "alerted": False
            })

# Szimulált logok (átalakítva parse‑után)
simulated_logs = [
    {"timestamp": "2026-05-07T10:00:00", "ip": "192.168.1.100"},
    {"timestamp": "2026-05-07T10:00:10", "ip": "192.168.1.100"},
    {"timestamp": "2026-05-07T10:00:20", "ip": "192.168.1.100"},
]
check_bruteforce(simulated_logs, threshold=3)
high_alerts = [a for a in alerts if a["alerted"]]
print("Brute‑force gyanús:", high_alerts)
```

**Gyakori hibák**  
- Nem szűrni az időablakot, így a múltbeli logokat is számolni.  
- Nem „IP‑alapú” csoportosítani, hanem egy globális számlálót használni.
