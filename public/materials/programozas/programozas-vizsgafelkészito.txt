#### 1. String normalization
**Rövid magyarázat:**  
Eltávolítjuk a felesleges szóközöket, kisbetűsítjük, ékezetes karaktereket lecseréljük. Így a keresés pontos lesz (pl. "Kovács" == "kovacs").

**Python-kódpélda:**
```python
import unicodedata

def normalize_string(text: str) -> str:
    # ékezetek eltávolítása
    text = unicodedata.normalize('NFKD', text).encode('ascii', 'ignore').decode('ascii')
    return text.strip().lower().replace(" ", "")

# Használat
print(normalize_string("Kovács János "))  # → "kovacsjanos"
```

**Tipikus vizsgafeladat:**  
Normalizáld a felhasználóneveket egy secrets.txt fájlból, és írd ki a normalizált verziót.

#### 2. Set használat O(1) lookuphoz
**Rövid magyarázat:**  
A lista keresés O(n), a set keresés O(1) → nagyon gyors duplikátum-ellenőrzés és lookup Secret Database-ben.

**Python-kódpélda:**
```python
secrets = set()
with open("secrets.txt", "r", encoding="utf-8") as f:
    for line in f:
        normalized = normalize_string(line.strip())
        if normalized in secrets:
            print(f"Duplikátum: {line.strip()}", file=sys.stderr)
        else:
            secrets.add(normalized)
```

**Tipikus vizsgafeladat:**  
Olvass be egy fájlt és gyűjtsd set-be a normalizált sorokat. Ha már létezik, írj stderr-re figyelmeztetést.

#### 3. File I/O és streaming
**Rövid magyarázat:**  
Nagy fájlokat ne olvass be egyszerre (`read()`), hanem soronként (`for line in file`) → memória-kímélő (streaming).

**Python-kódpélda:**
```python
def load_secrets(filename: str):
    secrets = set()
    with open(filename, "r", encoding="utf-8") as f:  # streaming!
        for line in f:
            normalized = normalize_string(line.strip())
            secrets.add(normalized)
    return secrets
```

**Tipikus vizsgafeladat:**  
Írj streaming módon fájlbeolvasót, ami 1 GB-os secrets fájlt is kezelni tud.

#### 5. bcrypt password hashing
**Rövid magyarázat:**  
Biztonságos jelszó-tárolás. Soha ne tárold plain text-ben! Bcrypt automatikusan sóz és lassú (brute-force ellen).

**Python-kódpélda:**
```python
import bcrypt

def hash_password(password: str) -> str:
    salt = bcrypt.gensalt(rounds=12)          # min. 10-12 rounds
    hashed = bcrypt.hashpw(password.encode(), salt)
    return hashed.decode()

def check_password(password: str, hashed: str) -> bool:
    return bcrypt.checkpw(password.encode(), hashed.encode())
```

**Tipikus vizsgafeladat:**  
Írj függvényt, ami bcrypt-tel hasheli a jelszót és elmenti fájlba.

### 1. String normalization Pythonban

**Rövid magyarázat**  
String normalization (pl. Unicode‑normalizálás) segít egységesített formába hozni a szöveget: pl. `é` és `é` különböző kanonikus formái egyforma normállá válnak, így biztonságosabb összehasonlítás / keresés. [docs.python](https://docs.python.org/3/library/unicodedata.html)

**Python‑példa**  

```python
import unicodedata

def normalize_text(s: str) -> str:
    return unicodedata.normalize("NFC", s.lower().strip())

plain = normalize_text("éÜlőkép  ")
accented = normalize_text("éÜLőkép")
print(plain == accented)  # True, ha a charok ténylegesen azonosak
```

**Vizsgafeladat típusok**  
- Írj függvényt, amely „egységes” keresési kulcsot készít felhasználói nevekből (case‑insensitive, ékezetek egységesítve).  
- Rendezzen egy lista neveket úgy, hogy `é` és `ê` ugyanannak számítson.  

**Gyakori hibák**  
- Nem normalizálni a szöveget, hanem csak a `.lower()`‑et használni, így különböző Unicode‑formák különkülünnek.  
- Nem kisbetűsíteni (`lower()` nélkül), így `User` és `user` különbözőnek látszik.  

**Ellenőrző kérdések**  
- Mi a különbség `NFC` és `NFD` között?  
- Miért fontos a normalizálás a CLI‑eszközök bejelentkezési / keresési logikájánál?  

**Források**  
- Python docs: `unicodedata.normalize` [docs.python](https://docs.python.org/3/library/unicodedata.html)
- Blog: „Unicode string normalization schemes in Python” [theniraj](https://theniraj.com/unicode-string-normalization-schemes-in-python)

### 2. set adattípus és O(1) keresés

**Rövid magyarázat**  
A `set` olyan adatstruktúra, amely csakis egyedi elemeket tárol, és a tartalmazás‑ellenőrzés (pl. `item in large_set`) gyakorlatilag konstans idejű, mert hash‑táblán alapszik. [stackoverflow](https://stackoverflow.com/questions/3949310/how-is-set-implemented)

**Python‑példa**  

```python
users = {"admin", "alice", "bob", "secret_user"}

def is_authorized(username: str) -> bool:
    return username in users  # O(1) keresés, függetlenül a set méretétől

print(is_authorized("admin"))   # True
print(is_authorized("eve"))     # False
```

**Vizsgafeladat típusok**  
- Döntsd el, hogy egy adott felhasználó része‑e egy nagy listában, de lineáris keresés helyett használj `set`-et.  
- Írj függvényt, ami két lista közös elemeit adja vissza (közös hozzáférési jogosultságok).  

**Gyakori hibák**  
- Minden keresést a `list`‑en végezni (`for x in lst`), így lineáris lesz a keresés, nem O(1).  
- Olyan típusokat adni a `set`‑be, amelyek nem hash‑elhetők (pl. `dict`, `list`).  

**Ellenőrző kérdések**  
- Miért nem lehet lista `set` eleme?  
- Miért a `set` jobb választás a hitelesített felhasználók tárolására, mint a `list`?  

**Források**  
- StackOverflow: „How is set() implemented?” [stackoverflow](https://stackoverflow.com/questions/3949310/how-is-set-implemented)
- Blog: „How Dictionary Lookup Operations Are O(1)” (a set ism.) [dev](https://dev.to/ajipelumi/how-dictionary-lookup-operations-are-o1-49pk)

### 3. Fájlbeolvasás nagyobb fájloknál, streaming szemlélet

**Rövid magyarázat**  
Nagy fájloknál nem érdemes egyszerre beolvasni az egész tartalmat, hanem soronként, „streaming” módon dolgozunk, így a memória‑igény nem nő a fájl méretével arányosan. [richardkorom](https://richardkorom.hu/python/fajlkezeles/)

**Python‑példa**  

```python
def count_lines_matching(filepath: str, pattern: str) -> int:
    count = 0
    with open(filepath, "r", encoding="utf-8") as f:
        for line in f:
            if pattern in line:
                count += 1
    return count

print(count_lines_matching("huge_secret_log.txt", "ERROR"))
```

**Vizsgafeladat típusok**  
- Olvasd be egy nagy logfájlt, és gyűjtsd össze a különböző IP‑ket egy `set`‑ben, anélkül, hogy a teljes fájlt memóriába töltenéd.  
- Írj függvényt, amely egy nagy kulcslista fájlból csak a releváns sorokat dolgozza fel.  

**Gyakori hibák**  
- `read()` helyett `readlines()` használata óriási fájlokra, ami teljesen elfogyasztja a memóriát.  
- Nem `utf‑8`‑as kódolást használni, így ékezetes karakterek széteshetnek.  

**Ellenőrző kérdések**  
- Miért nem jó a `f.read()` egy nagy logfájlhoz?  
- Milyen előnye van a `for line in f:`‑nek a `f.read().splitlines()`‑sal szemben nagy fájloknál?  

**Források**  
- Korom Richárd: „Fájlkezelés” [richardkorom](https://richardkorom.hu/python/fajlkezeles/)
- <info9>: „Fájlok olvasása és feldolgozása” [info9.edemmester](https://info9.edemmester.hu/hu/py/fo_open)

### 4. argparse CLI argumentumkezelés

**Rövid magyarázat**  
Az `argparse` modul segítségével egyszerűen, „szép” help‑szövegekkel és validációval lehet kezelni a CLI argumentumokat; ideális a Secret‑DB / Sentinel‑stílusú CLI eszközökhöz. [docs.python](https://docs.python.org/3/howto/argparse.html)

**Python‑példa**  

```python
import argparse

def main():
    parser = argparse.ArgumentParser(
        prog="secret_db",
        description="Secret Database CLI Tool"
    )
    parser.add_argument("command", choices=["add", "get", "list"])
    parser.add_argument("--user", required=True, help="Felhasználó")
    parser.add_argument("--secret", "-s", help="Titkos érték")
    parser.add_argument("--dry-run", action="store_true", help="Csak szimulál")

    args = parser.parse_args()
    print(f"Command: {args.command}")
    print(f"User: {args.user}")
    if args.secret:
        print("Secret: [REDACTED]")

if __name__ == "__main__":
    main()
```

**Vizsgafeladat típusok**  
- Írj olyan CLI‑t, amely `--action` és `--file` argumentumot fogad, ahol `--action` csak `import`, `export`, `verify` lehet.  
- Egészítsd ki a programot opcionális `--verbose` flaggel, amely logolja a műveleteket.  

**Gyakori hibák**  
- Nem `choices=`‑t használni, így a felhasználó nem megengedett opciókat is adhat meg.  
- `required=True` hiánya olyan argumentumoknál, ahol a program működéséhez elengedhetetlen.  

**Ellenőrző kérdések**

### 5. bcrypt jelszóhash‑elés

**Rövid magyarázat**  
A `bcrypt` erősen ajánlott jelszóhash‑elési algoritmus, lassú, iterációkon alapszik, így a brute‑force támadásokat jelentősen nehezíti. A `bcrypt.hashpw()` hash‑eli, a `bcrypt.checkpw()` a jelszó ellenőrzésére használható. [geeksforgeeks](https://www.geeksforgeeks.org/python/hashing-passwords-in-python-with-bcrypt/)

**Python‑példa**  

```python
import bcrypt

def hash_password(plain: str) -> bytes:
    salt = bcrypt.gensalt()
    return bcrypt.hashpw(plain.encode("utf-8"), salt)

def verify_password(plain: str, hash_bytes: bytes) -> bool:
    return bcrypt.checkpw(plain.encode("utf-8"), hash_bytes)

pwd = "Secret123!"
hashed = hash_password(pwd)
print("Verified:", verify_password("Secret123!", hashed))
```

**Vizsgafeladat típusok**  
- Írj hitelesítő függvényt, amely egy regisztrációs jelszót hash‑el és tárol, majd bejelentkezéskor ellenőrzi.  
- Egészítsd ki a CLI‑t egy `--hash` opcióval, amely egy jelszót a konzolról elfogadva mutatja a hash‑et.  

**Gyakori hibák**  
- Egy „fix” sót (`salt`‑ot) használni minden jelszóhoz, ami csökkenti a biztonságot.  
- A hasht `b""` formában próbálják meg változtatni stringként (pl. `str(hash)` és visszanyerni), ami nem működik.  

**Ellenőrző kérdések**  
- Miért nem tárolható a jelszó nyers formában a titkos adatbázisban?  
- Miért jó, ha a `bcrypt` lassú, nem pedig gyors?  

**Források**  
- GeeksforGeeks: „Hashing Passwords with Bcrypt” [geeksforgeeks](https://www.geeksforgeeks.org/python/hashing-passwords-in-python-with-bcrypt/)
- DEV.to: „Password hashing using Bcrypt in Python” [dev](https://dev.to/abbyesmith/password-hashing-using-bcrypt-in-python-2i08)

### 6. Environment variables Pythonban

**Rövid magyarázat**  
A környezeti változók (pl. `DATABASE_SECRET`, `DEBUG`) a `os.environ` vagy `os.getenv()` segítségével érhetők el; ideálisak titkos adatok (kulcsok, jelszavak) konfigurálására anélkül, hogy a kódban lennének. [sulipython.fandom](https://sulipython.fandom.com/hu/wiki/Adatbeolvas%C3%A1s)

**Python‑példa**  

```python
import os

def get_db_password() -> str:
    password = os.getenv("SECRET_DB_PASSWORD")
    if not password:
        raise ValueError("SECRET_DB_PASSWORD environment variable is not set.")
    return password

# Kivételkezelés nélkül:
# db_password = os.getenv("SECRET_DB_PASSWORD", "default_unsafe_password")
```

**Vizsgafeladat típusok**

**Gyakori hibák**  
- A jelszót a kódban „default”‑ként megadni (`os.getenv("X", "hardcoded_pw")`), ami biztonsági kockázat.

### 7. unittest alapok

**Rövid magyarázat**  
A `unittest` modul célja, hogy egyszerű, reprodukálható egység‑teszteket írhass kódodra; a `TestCase` osztályt tervezd úgy, hogy minden tesztmetódus önállóan futtatható legyen. [realpython](https://realpython.com/python-unittest/)

**Python‑példa**  

```python
import unittest
import hashlib

def file_hash(filepath: str) -> str:
    h = hashlib.sha256()
    with open(filepath, "rb") as f:
        while chunk := f.read(4096):
            h.update(chunk)
    return h.hexdigest()

class TestFileHash(unittest.TestCase):
    def test_empty_file(self):
        # tegyük fel, van egy empty.txt amiben nincs semmi
        h = file_hash("empty.txt")
        self.assertEqual(
            h,
            "e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855"
        )

    def test_nonexistent_file(self):
        with self.assertRaises(FileNotFoundError):
            file_hash("nonexistent_file.txt")

if __name__ == "__main__":
    unittest.main()
```

**Vizsgafeladat típusok**  
- Írj `unittest.TestCase`‑t a `hash_password()` / `is_authorized()` függvényekhez (pozitív + negatív tesztesetek).  
- Teszteld, hogy a CLI argumentumkezelés rossz opciókra helyesen dobjon `SystemExit`‑t.  

**Gyakori hibák**  
- Fixture‑t (pl. `setUp`) használni, de a tesztmetódusokat közvetlen futtatni, anélkül, hogy a `unittest.main()`‑t hívna.  
- A tesztek egymásra utalnak, így a sorrend befolyásolja az eredményt.
