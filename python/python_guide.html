Hogyan tanulj ebből az anyagból?

Stratégia, időbeosztás és hatékony tanulási módszerek

Kezdés előtt — 10 perc olvasás
🧠 A programozás tanulása más, mint a hagyományos tanulás
Nem elég "átolvasni" — a Python tanulása olyan, mint a gitár vagy a főzés: gyakorolni kell. Ez az útmutató úgy van felépítve, hogy 1/3 olvasás, 2/3 aktív kódolás legyen az arány.

Háromlépcsős tanulási ciklus minden fejezethez

1️⃣

Átolvasás

5-10 perc. Ne írj kódot, csak olvasd el a szekciót. Értsd meg, mit fogsz csinálni és miért.

2️⃣

Végigírás

15-30 perc. Írd le kézzel a kódot, ne másold be! Ez aktiválja az agyad motoros memóriáját.

3️⃣

Módosítás

20-40 perc. Változtass valamit! Adj hozzá új funkciót, változtasd meg a logikát. Ez a valódi tanulás.

Heti időbeosztási javaslat

📅 Heti struktúra (3×2 óra)
  • Hétfő + Szerda: Új anyag tanulása (1h olvasás + 1h gyakorlás)
  • Szombat: "Projekt nap" — csak a saját kis appodon dolgozol
  • Péntek: Heti gyakorlófeladatok megoldása
  • Vasárnap: Hét áttekintése, mi nem ment jól, hol akadtál el
🎯 Projekt nap szabályai
  • Válassz egy saját ötletet (pl. költségvetés nyilvántartó, címjegyzék, időjárás lekérdező)
  • Csak akkor nézz vissza az anyagba, ha elakadsz — próbáld megoldani magadtól!
  • A cél nem a tökéletes kód, hanem hogy használd amit tanultál
  • Vége: mentés GitHub-ra (ez fontos gyakorlás!)

Algoritmikus gondolkodás

Hogyan gondolkodj lépésekben? Pszeudokód és problémamegoldás

Alapozó — 30 perc
🤔 Mi az algoritmikus gondolkodás?
A számítógép nem "érzi", mit szeretnél — pontos, lépésről lépésre utasításokat kell adnod. Az algoritmikus gondolkodás az a képesség, hogy komplex feladatokat kisebb, végrehajtható lépésekre bontasz.

A 3 alapvető struktúra

➡️

Szekvencia

Egymás után következő lépések. A → B → C

Példa: kérj be számot → szorozd meg 2-vel → írd ki az eredményt

❓

Elágazás

Feltétel alapján döntés. HA X, AKKOR Y, KÜLÖNBEN Z

Példa: HA a szám > 0, AKKOR "pozitív", KÜLÖNBEN "negatív vagy nulla"

🔄

Ciklus

Ismétlés amíg feltétel teljesül. AMÍG X, CSINÁLD Y

Példa: AMÍG van elem a listában, CSINÁLD: számold meg őket

Pszeudokód: a tervrajz

✏️ Írj előbb pszeudokódot!
Mielőtt Python kódot írnál, fogalmazd meg szavakkal, mit akarsz. Ez segít kitisztítani a logikát, mielőtt belefutnál a szintaxis részleteibe.
Példa: maximum keresés pszeudokódban
# Feladat: Keresd meg a legnagyobb számot egy listában

1. Vegyük az első számot "maximumnak"
2. Menjünk végig a lista többi elemén:
   a. Ha az aktuális elem nagyobb, mint a "maximum":
      → Legyen ez az új "maximum"
3. A végén a "maximum" változóban lesz a válasz
Ugyanez Pythonban
def maximum_kereses(szamok):
    """Megkeresi a legnagyobb számot a listában."""
    if not szamok:          # Üres lista kezelése
        return None
    
    maximum = szamok[0]     # 1. lépés: első elem
    
    for szam in szamok[1:]: # 2. lépés: végigmegyünk a többin
        if szam > maximum:  # 2a. feltétel
            maximum = szam  # → új maximum
    
    return maximum          # 3. lépés: eredmény

# Teszt
print(maximum_kereses([3, 7, 2, 9, 1]))  # 9
print(maximum_kereses([-5, -2, -10]))     # -2

🎯 Gyakorló feladatok

1. Számok összegzése

Kezdő

Írj függvényt, ami egy számok listáját kapja és visszaadja az összegüket. Ne használd a beépített sum() függvényt!

Pszeudokód tipp: Indíts 0-ról, adj hozzá minden elemet egy ciklusban.

2. Páros számok szűrése

Kezdő

Készíts függvényt, ami kiszűri a páros számokat egy listából és visszaadja őket új listában.

Elágazás kell: HA szám % 2 == 0, AKKOR tedd az eredmény listába.

3. Szavak rendezése hossz szerint

Középhaladó

Adott egy szavak listája. Rendezd őket a legrövidebbtől a leghosszabbig! Használd a beépített sorted()-et egyedi kulcs függvénnyel.

Tipp: sorted(szavak, key=len) — a len függvény adja meg a rendezési kulcsot.
Valós projektben ez mire jó?

Az algoritmikus gondolkodás mindenre kell. Például:

  • E-commerce: keresd meg a legolcsóbb/legdrágább terméket a katalógusban
  • Adatfeldolgozás: szűrj ki érvénytelen e-mail címeket egy listából
  • Automatizálás: ismételd meg egy folyamatot minden fájlra egy mappában
  • Webfejlesztés: rendezd a kommenteket dátum szerint

Hibakeresés és hibák értelmezése

Mit csinálj, ha a kód nem fut? Traceback olvasás és debug stratégiák

Minden szinten — 25 perc
🚨 Fontos tudnivaló
A hibák normálisak. Még a senior fejlesztők is naponta látják ezeket. A különbség az, hogy ők tudják, hogyan olvassák el a hibaüzenetet és hogyan keressék meg a forrást.

Gyakori hibatípusok

Hiba neve Mit jelent? Tipikus ok Javítás
SyntaxError Szintaxis hiba — a Python nem érti a kódot Elfelejtett kettőspont (:), idézőjel, zárójel Ellenőrizd a hiba sora előtti sort is!
IndentationError Behúzási hiba Kevert szóköz és tab, vagy rossz behúzási szint VS Code-ban: "Convert Indentation to Spaces"
NameError Ismeretlen név Elgépelés, vagy nem létező változó használata Ellenőrizd a változónév helyesírását
TypeError Típushiba Pl. szám + string összeadása, vagy függvény hívás rossz típussal Ellenőrizd a változók típusát type()-tel
IndexError Indexelési hiba Lista túlindexelése (pl. 5 elemű listánál a [10]-edik elem) Ellenőrizd a lista hosszát előbb: len(lista)
KeyError Kulcs nem található Dict-ben nem létező kulcshoz próbálsz hozzáférni Használd: .get() vagy if kulcs in dict

Hibaelhárítás lépésről lépésre

  1. 1. Olvasd el a hibaüzenetet — alulról felfelé!

    A traceback az alján mutatja a konkrét hibát. A felső rész csak a "hívási lánc" (melyik függvény melyiket hívta).

    Traceback szerkezete
    Traceback (most recent call last):
  File "app.py", line 15, in <module>
    eredmeny = szamol(a, b)
  File "app.py", line 8, in szamol
    return x + y
TypeError: unsupported operand type(s) for +: 'int' and 'str'

# ↑ EZT A SORT OLVASD! Itt van a lényeg: int + str nem működik

  2. 2. Ellenőrizd a hiba sorát és az előtte lévőt

    Sokszor a hiba valójában egy sorral korábban van (pl. elfelejtett kettőspont), de Python csak később jelzi.

  3. 3. Használj print() debuggolási céllal

    Az egyik leghatékonyabb módszer: írd ki a változók értékét kulcsfontosságú pontokon.

    Print debug példa
    def szamol_osszeg(szamok):
    osszeg = 0
    for s in szamok:
        print(f"DEBUG: s = {s}, típusa = {type(s)}")  # ← IDE!
        osszeg += s
    return osszeg

# Futtatás:
szamol_osszeg([1, 2, "3"])  # TypeError-ra futna — a print megmutatja, hogy "3" string

  4. 4. Használd a VS Code debuggerét

    Break point (piros pötty) elhelyezése → F5 (Start Debugging) → változók vizsgálata futás közben.

🚫 Tipikus kezdő hibák debuggolásnál
  • "A kódom ugyanaz, mint a példában, de nem megy!" → Nézd meg szóköz-szinten. Python-ban a behúzás számít!
  • Elolvasni a hibát, de nem érteni → Másold be a hibaüzenetet Google-ba — valaki más már biztosan találkozott vele.
  • Túl sok változtatás egyszerre → Ha javítasz, mindig egy dolgot változtass, aztán tesztelj újra.
  • "Hátha majd magától elmúlik" → A hibák nem múlnak el maguktól. A szembenézés és megértés a tanulás része.
Elakadtál és nem érted a hibát?
  1. Írd le pontosan, mit próbálsz csinálni (1 mondatban)
  2. Másold ide a teljes traceback-et (ne csak az utolsó sort!)
  3. Másold ide a releváns kódrészletet (5-10 sor)
Ezzel a három infóval bárki tud segíteni, akár Stack Overflow-n, akár egy fórumon.

Mikor lépj a következő fázisra?

Átmeneti ellenőrzőpontok — biztos alapok a haladáshoz

Ellenőrzőlista minden fázis végére
🎯 A szabály: ne siess tovább, amíg ezek nem mennek!
A programozás építkező jellegű — ha az alapok lyukasak, a haladó részeknél összeomlás lesz. Használd ezt az ellenőrzőlistát minden fázis végén.
  • Tudod létrehozni és aktiválni egy virtuális környezetet (venv)
  • Tudod telepíteni a csomagokat pip-pel
  • Meg tudod nyitni a VS Code-ot és lefuttatni egy .py fájlt (nem csak a terminálban!)
  • Ismered a print(), input(), és az alapvető adattípusokat (str, int, float, bool)
  • Nem ijedsz meg egy traceback látványától — tudod olvasni a hibaüzenetet
Ha ez megvan: Átmehetsz Fázis 1-re! 🎉
  • Tudsz írni egy önálló programot 20-30 sorból (pl. számológép, kitalálós játék)
  • Érted az if/elif/else logikát és a feltételeket
  • Használni tudod a for és while ciklusokat
  • Ismered a lista és szótár alapműveleteit (hozzáadás, elérés, törlés)
  • Tudod kezelni a felhasználói inputot és validálni azt
Ha ez megvan: Megértetted a programozás alapvető logikáját! 🎉
  • Érted a referencia vs érték különbséget (miért változik a lista, ha "csak átadom"?)
  • Tudsz list comprehension-t írni
  • Tudsz dict-ben adatokat tárolni és onnan lekérdezni
  • Ismered a tuple, set alapokat és mikor használd őket
  • Tudsz egyszerű adattisztítást végezni (None kezelés, duplikátumok)
Ha ez megvan: Adatokkal dolgozol magabiztosan! 🎉
  • Tudsz saját függvényt írni paraméterekkel és visszatérési értékkel
  • Ismered az *args és **kwargs használatát
  • Tudsz fájlt olvasni és írni (txt, csv, json)
  • Használni tudod a try/except hibakezelést
  • Érted a scope-t (lokális vs globális változók)
Ha ez megvan: Moduláris kódot tudsz írni! 🎉
  • Tudsz saját osztályt írni __init__-tel és metódusokkal
  • Ismered az öröklést és a super() használatát
  • Tudod külön fájlokba szervezni a kódot és importálni őket
  • Írtál már legalább 3-5 egységtesztet pytest-tel
  • Érted a "dunder" metódusokat (__str__, __repr__, __eq__)
Ha ez megvan: Objektum-orientáltan gondolkodsz! 🎉

A haladó fázisoknál (pandas, ML, web, automatizálás) a cél nem az, hogy "tudj mindent", hanem hogy:

  • Tudod elolvasni a dokumentációt és értelmezni a példakódokat
  • Meg tudod keresni Stack Overflow-n a választ, és alkalmazni a saját kódodra
  • Be tudsz telepíteni új könyvtárakat (pip install) és importálni őket
  • Tudsz kis projekteket készíteni a tanult eszközökkel
  • Érted, mikor melyik eszközt érdemes használni (pl. pandas vs SQL)
Ha ez megvan: Önállóan tudsz tanulni és kutatni — ez a fejlesztői karrier kulcsa! 🎉
💡 Pro tipp: a "félig-meddig" szindróma

Ha úgy érzed, "félig érted" egy témát, ne menj tovább. Tölts el még egy-két órát gyakorlással, amíg magabiztos nem leszel. A "félig-meddig" tudás a haladó szakaszoknál visszaüt.

Jó jel, hogy készen állsz: Ha egy barátodnak el tudod magyarázni a koncepciót anélkül, hogy a jegyzetet néznéd.

Változók és adattípusok

Pythonban nem kell megadni a típust — a Python automatikusan kitalálja. Ezt dinamikus típusozásnak hívják.

valtozok.py
# ── Alap típusok ──────────────────────────────────────────
nev = "Anna"               # str  (szöveg)
kor = 25                   # int  (egész szám)
magassag = 1.72            # float (tizedes szám)
aktiv = True               # bool (igaz/hamis)
semmi = None               # NoneType (nincs érték)

# Típus lekérdezése
print(type(nev))           # 
print(type(kor))           # 

# ── Szöveg (str) műveletek ────────────────────────────────
gyumolcs = "alma"
print(gyumolcs.upper())    # ALMA
print(gyumolcs.capitalize())  # Alma
print(len(gyumolcs))       # 4
print(gyumolcs[0])         # a   (indexelés 0-tól)
print(gyumolcs[-1])        # a   (utolsó karakter)
print(gyumolcs[1:3])       # lm  (szeletelés)

# ── F-string formázás (Python 3.6+) ──────────────────────
# Ez a LEGJOBB módszer szöveg + változó kombinálásra!
nev = "Béla"
kor = 30
uzenet = f"Szia, {nev}! {kor} éves vagy."
print(uzenet)              # Szia, Béla! 30 éves vagy.

# Számítás az f-stringben is működik:
szam = 3.14159
print(f"Pi értéke: {szam:.2f}")   # Pi értéke: 3.14

# ── Számok ────────────────────────────────────────────────
a, b = 10, 3
print(a + b)    # 13  - összeadás
print(a - b)    # 7   - kivonás
print(a * b)    # 30  - szorzás
print(a / b)    # 3.333... - osztás (mindig float!)
print(a // b)   # 3   - egész osztás
print(a % b)    # 1   - maradék
print(a ** b)   # 1000 - hatványozás

💡 OOP – Egyszerűen elmagyarázva
Az OOP (Object-Oriented Programming) arról szól, hogy a kódot "tervrajzok" (osztályok) köré szervezzük. Ahogy egy "Autó" tervrajzból végtelen sok konkrét autót lehet gyártani, az osztályból is végtelen sok objektumot (példányt) lehet létrehozni.

# ── Alap osztály ──────────────────────────────────────────
class BankSzamla:
    """Egy egyszerű bankszámla osztály."""
    
    kamat = 0.02  # osztályváltozó: minden példányon ugyanaz
    
    def __init__(self, tulajdonos, egyenleg=0):
        """__init__ = konstruktor, amikor létrejön az objektum"""
        self.tulajdonos = tulajdonos   # példányváltozók
        self.egyenleg = egyenleg
        self._tranzakciok = []         # _ = "private" konvenció
    
    def befizet(self, osszeg):
        if osszeg > 0:
            self.egyenleg += osszeg
            self._tranzakciok.append(f"+{osszeg}")
            print(f"✅ Befizetve: {osszeg} Ft. Egyenleg: {self.egyenleg} Ft")
        else:
            print("❌ Az összeg pozitív kell legyen!")
    
    def kivesz(self, osszeg):
        if osszeg > self.egyenleg:
            print(f"❌ Nincs elég fedezet! (Van: {self.egyenleg} Ft)")
        else:
            self.egyenleg -= osszeg
            self._tranzakciok.append(f"-{osszeg}")
            print(f"💸 Kivéve: {osszeg} Ft. Egyenleg: {self.egyenleg} Ft")
    
    def kamatot_ir(self):
        kamat_osszeg = self.egyenleg * self.kamat
        self.egyenleg += kamat_osszeg
        print(f"💰 Kamat jóváírva: {kamat_osszeg:.2f} Ft")
    
    def __str__(self):
        """print() híváskor ezt adja vissza"""
        return f"Számla({self.tulajdonos}: {self.egyenleg} Ft)"
    
    def __repr__(self):
        return f"BankSzamla('{self.tulajdonos}', {self.egyenleg})"

import torch.optim as optim
from torch.utils.data import DataLoader, TensorDataset

X = torch.FloatTensor([[0,0], [0,1], [1,0], [1,1]])
y = torch.FloatTensor([[0], [1], [1], [0]])   # XOR

# ── 2. Modell definíció ───────────────────────────────────
class XORHalo(nn.Module):
    def __init__(self):
        super().__init__()
        self.reteget = nn.Sequential(
            nn.Linear(2, 8),      # 2 bemenet → 8 neuron
            nn.ReLU(),
            nn.Linear(8, 4),      # 8 → 4 neuron
            nn.ReLU(),
            nn.Linear(4, 1),      # 4 → 1 kimenet
            nn.Sigmoid()
        )
    
    def forward(self, x):
        return self.reteget(x)

# ── 3. Tanítás ────────────────────────────────────────────
model = XORHalo()
criterion = nn.BCELoss()
optimizer = optim.Adam(model.parameters(), lr=0.01)

for epoch in range(2000):
    y_pred = model(X)
    loss = criterion(y_pred, y)
    
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    
    if (epoch + 1) % 500 == 0:
        print(f"Epoch {epoch+1:4d} | Loss: {loss.item():.4f}")

# ── 4. Kiértékelés ────────────────────────────────────────
model.eval()
with torch.no_grad():
    predikciok = model(X)
    pred_osztaly = (predikciok > 0.5).float()
    
for inp, pred, cel in zip(X, pred_osztaly, y):
    print(f"Input: {inp.numpy()} → Pred: {int(pred.item())} (Cél: {int(cel.item())})")

def fajl_rendez(forras_mappa: Path):
    """Fájlokat rendez kiterjesztés szerint almappákba."""
    kategoriak = {
        "képek":     [".jpg", ".jpeg", ".png", ".gif", ".webp"],
        "dokumentumok": [".pdf", ".docx", ".xlsx", ".pptx", ".txt"],
        "kódok":     [".py", ".js", ".html", ".css", ".json"],
        "videók":    [".mp4", ".avi", ".mov", ".mkv"],
        "archívumok": [".zip", ".rar", ".tar", ".gz"],
    }
    
    atrakott = 0
    for fajl in forras_mappa.iterdir():
        if not fajl.is_file():
            continue
        
        kiterjesztes = fajl.suffix.lower()
        cel_mappa = None
        
        for kategoria, kiterjesztesek in kategoriak.items():
            if kiterjesztes in kiterjesztesek:
                cel_mappa = forras_mappa / kategoria
                break
        
        if cel_mappa:
            cel_mappa.mkdir(exist_ok=True)
            uj_hely = cel_mappa / fajl.name
            
            # Ha már létezik, adjunk dátumot a névhez
            if uj_hely.exists():
                datum = datetime.now().strftime("%Y%m%d_%H%M%S")
                uj_hely = cel_mappa / f"{fajl.stem}_{datum}{fajl.suffix}"
            
            shutil.move(str(fajl), str(uj_hely))
            print(f"✅ {fajl.name} → {cel_mappa.name}/")
            atrakott += 1
    
    print(f"\n🎉 {atrakott} fájl átrendezve!")

Változók, típusok, alap műveletek — Mélyebben

💡

Elakadtál? A változó "életciklusa"

Egy változó a Pythonban egy név, ami egy memóriában lévő objektumra mutat. Amikor írsz: x = 5, azt mondod: "x néven hivatkozom erre az 5 objektumra". Az x = 10 nem módosítja az 5-öt — x most egy másik objektumra mutat!

Dinamikus típusozás: előnyök és buktatók

Pythonban nem kell deklarálni a típusokat — a változó típusa az aktuálisan hivatkozott objektum típusától függ. Ez rugalmasságot ad, de rejtett hibákat is okozhat:

tipus_csapdak.py
# Csapda 1: Újrahasznált változónév más típussal
data = "100"      # string
# ... később a kódban ...
data = int(data)  # most int — de más kód még stringként kezelheti!

# Csapda 2: Referencia vs érték (későbbi részletesebben)
a = [1, 2, 3]
b = a             # b NEM másolat, hanem ugyanarra a listára mutat
b.append(4)       # ezért a is megváltozik!
print(a)          # [1, 2, 3, 4] — meglepő lehet kezdőként

# Csapda 3: Mutable vs Immutable
# int, str, tuple: immutable — "megváltoztatásuk" új objektumot hoz létre
x = 5
print(id(x))      # pl. 1402345678
x += 1            # x most új objektum!
print(id(x))      # más memóriacím — a 5 objektum változatlan maradt

📝 Mikor str vs int vs float?

  • str: szöveges adatok, azonosítók, címkék
  • int: számoláshoz, indexekhez, darabszámokhoz
  • float: mért értékek, tudományos számítások

🔤 String formázás: f-string vs .format()

  • f-string (ajánlott): olvasható, gyors, Python 3.6+
  • .format(): komplex sablonokhoz, régi kódokban
  • % formázás: elavult, kerüld!
⚠️ Tipikus hibák: Változók és típusok
  • if a == True: → helyette egyszerűen if a:
  • String konkatenáció loop-ban: result += item → helyette "".join(items) (hatékonyabb)
  • is vs == összekeverése: if x is 5: → használj ==-t érték-összehasonlításhoz!
  • Floating point pontatlanság: 0.1 + 0.2 == 0.3 → False! Használj decimal.Decimal-t pénzügyi számításokhoz

🎯 Gyakorlat: Változók és típusok

  • Írj egy programot, ami ellenőrzi egy változó típusát futás közben, és más-más műveletet végez aszerint könnyű
  • Készíts egy számológépet, ami kezeli a float pontatlanságot decimal modullal közép
  • Írj függvényt, ami tetszőleges mélységű listát (nested lists) lapít egy szintre közép

Referencia vs Érték másolás ezt sokan itt akadnak el

Ez a legfontosabb koncepció, amit meg kell érteni az adatszerkezeteknél:

referencia_vs_ertek.py
# ── Mutable objektumok (lista, dict, set) ─────────────────
# Ezeknél az = operátor referenciát ad, nem másolatot!

eredeti = [1, 2, 3]
referencia = eredeti          # ugyanazt a listát látják
eredeti.append(4)
print(referencia)             # [1, 2, 3, 4] — "meglepő"!

# Helyes másolás listáknál:
masolat1 = eredeti[:]         # szeleteléssel
masolat2 = list(eredeti)      # konstruktorral
masolat3 = eredeti.copy()     # metódussal
import copy
masolat4 = copy.deepcopy(eredeti)  # mély másolat (nested esetén!)

# ── Immutable objektumok (int, str, tuple) ────────────────
# Ezeknél nincs gond, mert "megváltoztatni" nem lehet őket

a = "hello"
b = a              # b is "hello"-ra mutat
a = a + " world"   # a most ÚJ string, b változatlan marad
print(b)           # "hello" — ez várható

Webfejlesztésnél gyakran kell JSON adatot feldolgozni. A json.loads() beolvasott adat dict-eket és listákat tartalmaz. Ha ezeket "biztonsági másolatként" módosítanád, de referenciát adsz át, az eredeti adat is módosul — és ezt debugging közben nehéz megtalálni!

⚠️ Tipikus hibák: Adatszerkezetek
  • Lista másolás nélküli átadás függvénynek, ami módosítja → a hívónál is megváltozik!
  • dict[key] helyett dict.get(key) használata hiányzó kulcsoknál → KeyError elkerülése
  • Lista módosítása iterálás közben → használj list comprehension-t vagy másolatot!
  • [] vs list() — nincs nagy különbség, de {} üres dict, nem set! Set-hez: set()

🔍 Haladó: List Comprehension vs Generator Expression

▼

A list comprehension ([x for x in items]) azonnal létrehozza az egész listát a memóriában. A generator expression ((x for x in items)) "lustán" adja az elemeket egyesével — nagy adatmennyiségnél ez kritikus lehet:

# 10 millió elem — list comprehension (sok memória!)
negyzetek = [x**2 for x in range(10_000_000)]

# Ugyanennyi elem — generator (szinte 0 memória)
negyzetek_gen = (x**2 for x in range(10_000_000))

# A generator csak akkor számít, amikor kell:
for n in negyzetek_gen:
    if n > 100:
        break  # itt abbahagyjuk, nem pazaroltunk memóriát

haladó A generátorokat függvényekben a yield kulcsszóval is készítheted.

🎯 Gyakorlat: Adatszerkezetek

  • Implementálj egy LRU (Least Recently Used) cache-t dict és OrderedDict segítségével közép
  • Írj függvényt, ami két dict-et rekurzívan merge-öl (mély merge) közép
  • Készíts egy egyszerű adatbázis indexet set-ekkel, gyors kereséssel nehezebb
  • Írj egy programot, ami megtalálja két szöveg közös karaktereit halmazműveletekkel könnyű

A break, continue, else trió kevésbé ismert

Pythonban a ciklusoknak van else ága is — ez akkor fut le, ha a ciklus NEM break-kel ért véget:

ciklus_else.py
# Prímkeresés — ciklus-else használata

def is_prime(n):
    if n < 2:
        return False
    for i in range(2, int(n**0.5) + 1):
        if n % i == 0:
            print(f"{n} osztható {i}-vel")
            break  # találtunk osztót → nem prím
    else:
        # Ez csak akkor fut, ha a for NEM break-elt
        print(f"{n} prímszám!")
        return True
    return False

# Gyakorlati példa: keresés lista-ban
def find_user(users, name):
    for user in users:
        if user["name"] == name:
            return user  # megtaláltuk!
    else:
        # Nem találtuk meg
        return None
⚠️ Tipikus hibák: Vezérlési szerkezetek
  • while i < len(lista): → helyette for item in lista: (Pythonic!)
  • Végtelen ciklus while i > 0: de i sosem változik → mindig legyen kilépési feltétel
  • Lista módosítása for-ral: for x in l: l.remove(x) → soha! Másolatot iterálj: for x in l[:]:
  • Elfelejtett : a ciklus/if végén → SyntaxError

  • Írj "jelszó-erősség" ellenőrzőt, ami for/while/break/continue kombinációt használ könnyű
  • Készíts számkitaláló játékot, ahol a while-else szerkezetet használod a nyerés/vesztés jelzésére közép
  • Implementálj egy retry mechanizmust: max 3 próbálkozás, hiba esetén várj és próbáld újra közép

# ── Lista unpacking híváskor ───────────────────────────────
def osszeg(a, b, c):

🔍 Haladó: Lambda és First-Class Functions

▼

Pythonban a függvények "first-class citizens" — változókban tárolhatod őket, argumentumként adhatod át, visszatérési érték lehet:

# ── Függvény argumentumként ──────────────────────────────
operaciok = {
    "+": lambda a, b: a + b,
    "-": lambda a, b: a - b,
    "*": lambda a, b: a * b,
}

def szamologep(a, b, op):
    return operaciok.get(op, lambda a, b: None)(a, b)

# ── Closure — függvénygyár ─────────────────────────────────
def szorzo_fuggveny(faktor):
    def szoroz(n):
        return n * faktor  # faktor "megjegyezve" a closure-ból
    return szoroz

ketszerezo = szorzo_fuggveny(2)
hatszorozo = szorzo_fuggveny(6)
print(ketszerezo(5))   # 10
print(hatszorozo(5))   # 30

haladó A closure-ök és first-class functions a dekorátorok alapját képezik.

🎯 Mikor használj lambda-t?

  • Egyszerű, egy-soros függvények
  • sorted(), map(), filter() key függvények
  • GUI callback-ek (egyszerű esetek)
  • Nem: komplex logikára — írj rendes függvényt!

📋 Mikor használj def-et?

  • Többsoros logika
  • Docstring szükséges
  • Újrafelhasználható komponens
  • Testelhető egység
🌍 Valós projektben ez mire jó?

A *args és **kwargs wrapper függvényeknél elengedhetetlen — pl. caching decorator, logging, vagy API hívások retry mechanizmusa. A functools.wraps megtartja az eredeti függvény metaadatait.

⚠️ Tipikus hibák: Függvények
  • Mutable default argumentum: def f(l=[]): → minden hívás ugyanazt a listát látja! Használj None-t
  • lambda ciklusban: [(lambda: i) for i in range(3)] → minden lambda ugyanazt az i-t látja!
  • Rekurzió mélység: Python alapból 1000 rekurzív hívásig enged → nagy adathoz iteratív megoldás
  • return nélkül a függvény None-t ad vissza → ellenőrizd, hogy minden ágon van return!

A Context Manager (with) működése

A with kulcsszó nem csak szintaktikai cukorka — garantálja az erőforrások felszabadítását, még hiba esetén is:

context_manager.py
# ── Mi történik a with mögött? ─────────────────────────────
# Hagyományos (rossz) módszer:
f = open("file.txt")
data = f.read()      # ha itt hiba van, a fájl nyitva marad!
f.close()            # sosem fut le

# With (jó) módszer — garantált f.close():
with open("file.txt") as f:
    data = f.read()  # ha hiba van, f.close() MÉG MINDIG lefut!

# ── Saját context manager ─────────────────────────────────
from contextlib import contextmanager

@contextmanager
def managed_resource(name):
    print(f"{name} inicializálás...")
    resource = {"name": name, "status": "ready"}
    try:
        yield resource  # itt adja át a vezérlést
    finally:
        print(f"{name} takarítás...")  # mindenképp lefut

with managed_resource("adatbázis") as db:
    print(f"Használom: {db}")
    # raise ValueError("Hiba!")  # ha hiba van, a finally így is lefut

# Kimenet:
# adatbázis inicializálás...
# Használom: {'name': 'adatbázis', 'status': 'ready'}
# adatbázis takarítás...

Hibakezelés hierarchia és best practice

hiba_kezeles.py
# ── Specifikus → Általános sorrend ────────────────────────
# JÓ: konkrét kivételeket először

try:
    process_data()
except FileNotFoundError as e:           # Konkrét: fájl nem létezik
    logger.error(f"Fájl hiányzik: {e}")
    raise CustomDataError("Adat nem elérhető") from e
except PermissionError:                   # Konkrét: nincs jog
    logger.error("Nincs jogosultság")
    # fallback működés...
except OSError as e:                      # Általánosabb: bármilyen OS hiba
    logger.error(f"Rendszerhiba: {e}")
except Exception as e:                    # Minden egyéb (óvatosan!)
    logger.critical(f"Váratlan hiba: {e}")
    raise  # újradobjuk, ne nyeljük el!

# ── EAFP vs LBYL ───────────────────────────────────────────
# EAFP: "Easier to Ask Forgiveness than Permission" (Pythonic!)
try:
    value = my_dict["kulcs"]  # megpróbáljuk
except KeyError:
    value = "alapérték"       # kezeljük a hibát

# LBYL: "Look Before You Leap" (kevésbé Pythonic)
if "kulcs" in my_dict:       # előbb ellenőrizzük
    value = my_dict["kulcs"]
else:
    value = "alapérték"

# Pythonban az EAFP preferált — gyorsabb és race-condition mentesebb
🌍 Valós projektben ez mire jó?

API fejlesztésnél (FastAPI) a try/except struktúra határozza meg, hogy a kliens 500-as hibát kap-e, vagy szép JSON hibaüzenetet megfelelő HTTP státuszkóddal (400, 404, 422). A raise ... from ... láncolás megőrzi az eredeti stack trace-et debugginghoz.

⚠️ Tipikus hibák: Fájl/Hibakezelés
  • Nyers except: vagy except Exception: → elnyel minden hibát, beleértve KeyboardInterrupt-ot is!
  • except: blokkban pass → a hiba eltűnik, senki sem tud róla — legalább logold!
  • Fájl encoding figyelmen kívül hagyása → mindig adj meg encoding="utf-8"-at!
  • JSON-t stringként kezelni eval()-lel → biztonsági rés! Használj json.loads()-t

🎯 Gyakorlat: Fájlkezelés és hibakezelés

  • Írj egy fájlmásolót, ami kezeli a létező fájl felülírását, jogosultsági hibákat közép
  • Készíts JSON konfiguráció-kezelőt validációval és értelmes hibaüzenetekkel közép
  • Implementálj log rotációt: 1 MB-os log fájlok, max 5 darab archívum nehezebb

Objektumorientált programozás — OOP alapokon

🧠

Mi az OOP lényege valójában?

Az OOP nem „bonyolultabb szintaxis" — hanem kódorganizáció. Az adatokat (attribútumokat) és a műveleteket (metódusokat) egy egységbe, az objektumba zárjuk. Ez elrejti a bonyolultságot (encapsulation) és lehetővé teszi az újrafelhasználást (inheritance, composition).

Composition vs Inheritance

A klasszik OOP tananyag sokat beszél az öröklésről (inheritance), de modern Pythonban a composition preferált:

composition_vs_inheritance.py
# ── Inheritance (IS-A kapcsolat) ─────────────────────────
# "Egy kutya IS egy állat" — de csak egy szülő lehet Pythonban!

class Animal:
    def move(self):
        pass

class Dog(Animal):  # Dog IS-A Animal
    def bark(self):
        return "Woof!"

# ── Composition (HAS-A kapcsolat) ─────────────────────────
# "Egy autó HAS-A motor" — rugalmasabb, több komponens is lehet

class Engine:
    def start(self):
        return "Engine running"

class Wheels:
    def rotate(self):
        return "Wheels spinning"

class Car:
    def __init__(self):
        self.engine = Engine()    # Car HAS-A Engine
        self.wheels = Wheels()  # Car HAS-A Wheels
    
    def drive(self):
        return f"{self.engine.start()} and {self.wheels.rotate()}"

# ── A composition előnyei ─────────────────────────────────
# 1. Több "szülő" hatás: Car használ Engine-t ÉS Wheels-t
# 2. Futás közben is cserélhető a komponens
# 3. Tesztelésnél könnyebb mock-olni
# 4. Nincs "fragile base class" probléma

# ── Dependency Injection példa ───────────────────────────
class Service:
    def __init__(self, db_connection, logger):
        self.db = db_connection  # injektált függőség
        self.logger = logger
    
    def process(self):
        self.logger.info("Processing...")
        return self.db.query("SELECT * FROM data")

🧬 Mikor használj öröklést?

  • Valódi IS-A kapcsolat (kutya = állat)
  • Interface/abstract base class implementálás
  • Közös kódmegosztás szorosan kapcsolódó osztályok között
  • Method override konkrét viselkedéshez

🔗 Mikor használj composition-t?

  • Objektumok összeállítása független komponensekből
  • Több "képesség" hozzáadása (mixin helyett)
  • Runtime konfigurálhatóság szükséges
  • Unit tesztelhetőség fontos
🌍 Valós projektben ez mire jó?

Egy FastAPI alkalmazásban az BaseService osztályból való öröklés biztosítja, hogy minden service ugyanúgy loggol és kezeli a hibákat. A composition használatával a UserService egy EmailClient-et és Database-et használ anélkül, hogy azok implementációját ismerné — dependency injection segítségével.

⚠️ Tipikus hibák: OOP
  • self elfelejtése metódus paramétereknél → TypeError: method() takes 0 positional argument...
  • Példányváltozó vs osztályváltozó keverése → osztályváltozót minden példány látja, ez lehet nem kívánt!
  • super() elfelejtése override-olt metódusban → a szülő logikája nem fut le
  • Túl mély öröklési lánc (3+ szint) → a kód nehezen követhető, preferáld a composition-t!

🎯 Gyakorlat: OOP

  • Refaktorálj egy procedural scriptet OOP-re: azonosítsd az entitásokat és viselkedéseket közép
  • Írj @dataclass-t használó modelleket, majd konvertáld át Pydantic modellekké közép
  • Implementálj egy egyszerű plugin rendszert composition-nel nehezebb
  • Készíts ABC-t (Abstract Base Class) repository pattern-hez nehezebb

Vectorizáció: miért olyan gyors a NumPy?

A NumPy C-ben írt, optimalizált rutinokat használ. A vectorizált műveletek (pl. array * 2) C-szinten párhuzamosítva futnak. A Python for-loop (pl. [x*2 for x in lista]) sokkal lassabb, mert minden iterációnál Python interpreter overhead van.

pandas_profi.py
# ── Pandas anti-patterns és helyes megoldások ─────────────

# ❌ ROSSZ: iterrows — nagyon lassú!
for idx, row in df.iterrows():
    if row['age'] > 18:
        df.loc[idx, 'adult'] = True

# ✅ JÓ: vectorizált művelet
df['adult'] = df['age'] > 18

# ── apply vs vectorizált függvények ───────────────────────
# ❌ apply — Python szinten fut

df['result'] = df['value'].apply(lambda x: x**2)

# ✅ NumPy vectorizált függvény (gyorsabb)
import numpy as np
df['result'] = np.square(df['value'])

# ── Groupby átalakítások ──────────────────────────────────
# Több aggregáció egyszerre
stats = df.groupby('category').agg({
    'price': ['mean', 'min', 'max', 'count'],
    'quantity': 'sum'
}).round(2)

# Column flattening (többszintű oszlopnevek egyszerűsítése)
stats.columns = ['_'.join(col).strip() for col in stats.columns.values]

# ── Merge/join típusok ────────────────────────────────────
# inner: csak közös kulcsok (default)
# left: minden bal oldali sor
# right: minden jobb oldali sor
# outer: minden sor mindkét táblából

⚠️ Tipikus hibák: NumPy/pandas
  • DataFrame módosítása iterálás közben → RuntimeError, copy() vagy vektorizált művelet!
